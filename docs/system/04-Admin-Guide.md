# EDU-TRACK — Administrator Guide

This guide is for school administrators who manage EDU-TRACK users, configuration, and overall data quality.

## 1. Getting started

### Step 1: Sign in

- Open the EDU-TRACK web address provided by your school IT team.
- Enter your username/email and password.
- If you cannot sign in, contact the system administrator at the school.

### Step 2: Confirm your role

- Confirm your account has the Admin role/permissions.
- If you can sign in but cannot access administrative pages, request an Admin role assignment.

## 2. Initial school setup (first-time deployment)

### Step 1: Confirm school profile/settings

- Set the school name and basic profile details.
- Confirm timezone and academic year conventions.

### Step 2: Create terms/semesters

- Create term/semester records for the current academic year.
- Confirm start/end dates.
- Set the active term.

### Step 3: Configure classes/streams

- Create classes (e.g., Form 1, Grade 6).
- Create streams where applicable (e.g., East/West).
- Assign class teachers if the system supports it.

### Step 4: Confirm subjects and grading structure

- Create subjects.
- Confirm assessment types (CATs, Midterm, Endterm) if used.
- Confirm grading rules if your deployment supports automated grades.

## 3. User management (staff accounts and roles)

### Step 1: Create staff accounts

- Add new staff users (teachers, finance officers, staff).
- Ensure the correct phone/email is captured if notifications are used.

### Step 2: Assign roles and permissions

Recommended role model:

- **Admin:** full configuration and user management
- **Teacher:** attendance and assessments
- **Finance:** fee management and statements

### Step 3: Offboarding (exiting staff)

- Disable the staff account immediately.
- Transfer responsibilities (e.g., class assignments) to another staff member.

## 4. Student admissions and enrollment

### Step 1: Register learners

- Add students individually or import where the system supports it.
- Ensure admission numbers are unique.

### Step 2: Add parent/guardian contacts

- Capture at least one guardian contact per learner where possible.
- Validate phone numbers for SMS notifications.

### Step 3: Assign class/stream

- Assign each learner to the correct class/stream for the active term.
- Confirm lists are correct before teachers start attendance and marks entry.

## 5. Finance configuration (admin-side)

### Step 1: Define fee items and fee structure

- Create fee items (tuition, lunch, transport, etc.).
- Define a termly fee structure per class/grade if your deployment supports it.

### Step 2: Confirm finance users and access

- Ensure only authorized finance staff can view balances and statements.

## 6. Reports and routine checks

### Step 1: Weekly checks

- Confirm attendance completion rates.
- Review recent changes (new admissions, class moves).
- Check for duplicates (learners/users).

### Step 2: Termly checks

- Confirm all assessment entries are complete before report generation.
- Export/backup key term reports as required by the school.

## 7. Security and operational best practices

### Step 1: Password and access hygiene

- Enforce strong passwords.
- Do not share admin credentials.
- Remove or disable unused accounts.

### Step 2: Backups (school-hosted)

- Ensure automated daily database backups are configured.
- Retain backups according to school policy.
- Perform restore drills at least quarterly.

## 8. Getting help (support and escalation)

When escalating a problem, provide:

- what you were doing
- the exact error message
- time of occurrence
- affected users/roles
